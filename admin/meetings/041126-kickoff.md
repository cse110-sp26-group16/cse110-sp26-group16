### 4. Team Norms & Ground Rules
Topics to discuss for the team contract:
- How will we handle getting assignments done on time?
    - Set hard deadlines for all of us, check in with each other throughout the week, quick talks on Tuesday/fridays after class, check in on Fridays before the weekend
    - Stay connected with each other and always be communicating, messaging and being active in slack, sending mid week updates in slack
    - Not waiting until the last second, do assignments early
- How will we handle disagreements?
    - Try to make sure it never gets personal, try to get like another perspective in the loop and see if there is any compromise anywhere
- How will we handle a teammate who isn't pulling their weight?
    - Possibly switch the roles, switch to something that they are better at / enjoy doing more
    - Don't push too hard but motivate
    - Be opening to adjusting the workload between the group
- Code review expectations (e.g., 1 reviewer required before merge)
    - Pull requests with 1 reviewer before merge
- Branching strategy
    - Add issues on github repo, use issues for branches
    - "your_name/(feature/bug/improvement)/short_desc", ex. yuval/feature/nav-bar
    - "Alec/99/bug" -> name/issue#/type
- Definition of "done"
    - Once the whole pull request is reviewied and completely merged, can be done with the issue
